DOORS: Paradox Building Rules

Contents

    Building Standards for DOORS: Paradox

    Everything here is a requirement. If a build doesn't meet these rules, the room will NOT be added to the game. Keep scale, distance, assets, and hierarchy consistent across all rooms.

    Under no circumstance should you ignore these rules, if these rules are too difficult to understand for you, you might not be suitable to be a builder for Paradox, this is not being rude, this is enforcing quality.

    Basics

    #

    Grid & Alignment
    • All positions and scales must be multiples of 0.125 studs. (…0.125, 0.25, 0.5, 1.0, etc.)
    • Never use free-scale/floating values like 0.373 unless explicitly called out in the spec for a prop.
    • Pivots anchored to logical center or bottom of assets.
    • No z-fighting and no micro-nudges to look right to the eye. Make the geometry right, it'll all work out at the end.

    When “off-grid” is allowed
    • Decor only: subtle rotation or offset of furniture to break repetition.
    • Never for structure (walls, floors, ceilings, trims, pillars, door/window frames).

    Room Basics

    #

    Start & Exit orientation
    • Start part faces outside of the main room.
    • Exit part faces inside of the main room.
    • Both must be on-grid and flush with door thresholds; no micro gaps.

    RoomEntity_SpawnNode
    • Required in every room.
    • Place in front of Start or roughly center of the room, unobstructed.
    • Keep 2x2x2 studs of clear volume around it; no stuff within that area, think of this as like the nodes personal space.

    Waypoints
    • Create the top-level folder: Waypoints.
    • Only required if Start and Exit are not line-of-sight.
    • Inside Waypoints, add one folder per exit: 1, 2, …
    • Drop waypoint parts in order inside each subfolder. Keep them named naturally or numbered.

    Folders

    #

    Required tree

     
    Room Model
      ├─ Assets          - structure: Walls, Floor, Ceiling, Windows/Frames
      ├─ Furniture       - anything with a Part named "Primary" inside the model
      ├─ Lights          - all light sources (bulbs, sconces, strips)
      ├─ RoomHitbox      - convex/union set that fully contains the room
      ├─ Scriptables     - gates, triggers, interactions
      ├─ Waypoints       - path folders (1, 2, ...)
      ├─ RoomModule      - ModuleScript: enter/exit hooks (remove if unused)
      ├─ Start           - entry locator (faces in)
      ├─ Exit            - exit locator (faces out)
      └─ RoomEntity_SpawnNode
    • Do not rename these core folders/objects.
    • Structure inside Assets can be further sub-organized, but keep it sane.

    Furniture rule
    • Any model containing a Part named Primary is classified as Furniture and must live in Furniture.
    • Use the Multitool to convert raw models into Furniture models.

    Lighting

    Drag & drop prefabs; place where they look best, but follow the guardrails.
    #

    Non-negotiables
    • Do not change light colors or any internal parts/materials/names. The lights are replaced at runtime, edits will break lighting systems and make the room look wrong in game
    • Allowed edits: position, rotation, duplication. Not allowed: scale, recolor, re-mesh, re-parent, PrimaryPart changes.
    • All light models live in the room's Lights folder.

    General placement
    • Baseline spacing between light bulbs is 20 studs. You can choose whatever looks best.
    • LightBulb (main) / RoofLight: typically 3.5 studs below the finished ceiling.
    • Wall lights (WallLight, MetalWallLight, lanterns): centered on panels/pillars; keep tops level down a corridor.
    • Other lights (HugeLight, TowerLight, Floodlight_Stand): use as accents/fills; avoid over-crowding.

    Walls

    Template use, symmetry, heights, and bounds.
    #

    Floor plan grid (15-stud rooms)

    Do

    • Build floor plans in 10-stud steps (10, 20, 30, and so on).
    • Think in 10x10 “cells” and combine whole cells for the room size.

    Don't

    • Use sizes like 42 or 100 studs, they break pillar and wall spacing.
    • Eyeball the plan without snapping to the 10-stud rhythm.

    Centered entrances: equal left/right walls
    • If an Entrance/Exit inset sits in the middle of a longer wall, the wall segments to the left and right of it must be the same width.
    • Example: if the right segment to the next corner/wall break is 10 studs, the left segment must also be 10 studs.

    Floors
    • Every single floor in every room when using the default height for each room should be 13.75 studs tall, past that a new floor should be made unless otherwise, no floor to ceiling height should be lower than 13.75 studs

    Taller walls
    • To make walls taller, only scale the main wall panel upward (Y). Do not duplicate the baseboard.
    • Two-floor rooms only: if there are two distinct floors, each floor has its own baseboard at that floor’s height and it runs around the whole room.

    Meet the floor exactly
    • Walls must sit exactly on the floor bounds. No floor sticking out under/around the wall.
    • One big floor part is okay for multi-room sets, but still keep walls aligned to its exact edges.

    Pillars

    Standard offsets from walls and corners.
    #

    Spacing
    • Every pillar sits exactly 10 studs away from eachother, pillars are spaced 10 studs away from other pillars. Not closer, not farther.
    • This goes for all pillars, if the last pillar of the room is following the 10 studs away, but is not exactly meeting the corner of the room properly, then you are NOT making the room correct, and you will need to change the floor and walls.

    Offset from a straight wall
    • Every pillar sits exactly 0.375 studs away from the wall. Not closer, not farther, unless it's a corner.
    • Measure from the pillar's closest face to the wall's face.

    Pillars in corners
    • Corner pillars are 1.25 studs away from each wall (X and Z). That makes the visible footprint exactly 1.25 x 1.25. (Y can be whatever the room needs.)
